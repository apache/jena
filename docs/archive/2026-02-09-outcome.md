## What Changed

### Phase 0: Bug Fixes
1. **`updateDocument()` now calls `facetsConfig.build(doc)`** — matching the `addDocument()` pattern
2. **`getFacetCounts()` uses `TermInSetQuery`** instead of `BooleanQuery` — avoids `TooManyClauses` with large result sets

### Phase 1: API Split
Replaced 3 PFs with 2:

| Before | After |
|--------|-------|
| `text:query` | `text:query` (now accepts JSON filter arg) |
| `text:queryWithFacets` | Removed |
| `text:facetCounts` | `text:facet` (JSON array/object syntax) |

### New SPARQL Syntax

```sparql
# text:query with filters
(?s ?score) text:query ("climate" '{"category": ["Technology"]}' 20)

# text:facet with facet fields and filters
(?f ?v ?c) text:facet ("climate" '["category", "author"]' '{"category": ["Technology"]}' 10)
```

### Filter Semantics
- JSON object: `'{"field": ["val1", "val2"]}'`
- Values within a field: OR'd
- Across fields: AND'd
- Detected by `{` prefix (unambiguous vs query strings, URIs, integers)

### Shared Execution
`SearchExecution` class stores lazy hits + facet counts in `ExecutionContext`. When `text:query` and `text:facet` use matching params, the Lucene query runs once.

## Manual Fuseki Testing

Build and run:
```bash
mvn clean install -pl jena-fuseki2/jena-fuseki-server -am -DskipTests

java -jar jena-fuseki2/jena-fuseki-server/target/jena-fuseki-server-6.0.0-SNAPSHOT.jar \
    --config ~/fuseki-facet-test/config.ttl
```

Test queries (after loading data):
```bash
# text:facet
curl -X POST "http://localhost:3030/ds" \
    -H "Content-Type: application/sparql-query" \
    -d 'PREFIX text: <http://jena.apache.org/text#>
SELECT ?f ?v ?c WHERE {
  (?f ?v ?c) text:facet ("learning" '\''["category", "author"]'\'' 10)
} ORDER BY ?f DESC(?c)'

# text:query with filter
curl -X POST "http://localhost:3030/ds" \
    -H "Content-Type: application/sparql-query" \
    -d 'PREFIX text: <http://jena.apache.org/text#>
SELECT ?s ?score WHERE {
  (?s ?score) text:query ("learning" '\''{"category": ["technology"]}'\'' 20)
} ORDER BY DESC(?score)'
```
