About This Demo

This demo showcases SHACL-based entity indexing with faceted search in Apache Jena Fuseki. The dataset contains 21 entities across 3 types (Mining Reports, Boreholes, Sites) indexed using Lucene. Each feature below links to the page where you can see it in action.

Full-Text Search

Lucene-backed text search across entity labels and descriptions. All 21 entities are indexed across 3 types. Matching is ranked by relevance score.

Try: search for “copper”  or  search for “exploration”

SHACL-Based Entity Indexing

Entities are indexed as whole documents using SHACL shapes (text:shapes), not individual triples. Each shape targets a class (MiningReport, Borehole, Site) and defines which properties to index as typed fields.

See: Configuration page

Faceted Navigation

Filter results by keyword field values. Facet counts update dynamically as you refine your search. There are 6 facetable fields: entityType, commodity, state, operator, status, and authorName.

Try: click any facet checkbox on the Search page

Multi-Valued Fields

Fields like commodity support multiple values per entity. The facet counts reflect all values, so a single entity can appear under several facet buckets.

Try: filter by commodity “Copper” — Olympic Dam has commodities Copper, Uranium, Gold

Typed Fields

Three field types control how values are indexed and queried: TextField for full-text search, KeywordField for exact-match facets, and IntField for numeric, sortable values. The demo data has year and depth as IntFields.

See: Configuration page field type column

Sequence Paths

SHACL property paths that traverse multiple predicates. Two examples in the config:

  • entityType: rdf:type / rdfs:label — resolves the type URI to its human-readable label (“Mining Report”, not a URI)
  • authorName: ex:authoredBy / ex:name — indexes the author’s name on the report entity

Try: filter by authorName “Dr Sarah Jones” — shows her reports with resolved name

Inverse Paths

sh:inversePath traverses a predicate backwards. authoredByUri uses ^ex:authored to index which authors wrote a report, starting from the Author entity’s ex:authored link.

See: Configuration page, MiningReportShape

Combined Search + Facets

luc:query and luc:facet execute in a single SPARQL UNION query, sharing the same Lucene search via SearchExecution. This avoids redundant index lookups.

Watch: the SPARQL Log panel shows both result retrieval and facet counting in one query

SPARQL Log

Live panel showing every SPARQL query sent to Fuseki. Shows the actual luc:query and luc:facet property functions with filter JSON. Newest queries appear at the top.

See: right-hand panel on the Search page

Configuration Viewer

Parses config.ttl directly in the browser using N3.js (an RDF/Turtle parser). Shows shapes, fields, types, and settings without a server-side API.

See: Configuration page
