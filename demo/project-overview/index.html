Full-Text Search with Filters

luc:query extends Jena text search with JSON filter arguments and a ?totalHits binding. Filters are AND across fields, OR within a field. 

PREFIX luc:  <urn:jena:lucene:index#>

SELECT ?entity ?score ?totalHits
WHERE {
    (?entity ?score ?_lit ?totalHits)
        luc:query ('copper' '{"state":["QLD"]}' 10)
}
ArgumentTypeDescription
queryStringStringLucene query (supports wildcards, phrases, field scoping)
filterJSON objectStructured facet filters — {"field": ["val1","val2"]}
limitIntegerMax result rows. ?totalHits still returns full count
Key Innovation
?totalHits is computed via IndexSearcher.count() — an O(1) operation that doesn't retrieve documents. This enables "Showing 10 of 1,234 results" without fetching all rows.
Filter Semantics
Within a field: OR (matches any value)
Across fields: AND (must match all)

Implemented via Lucene TermInSetQuery wrapped in BooleanQuery.

Faceted Navigation

luc:facet returns aggregate counts per field value. Powered by Lucene's SortedSetDocValuesFacetCounts for O(1) counting. 

PREFIX luc:  <urn:jena:lucene:index#>

SELECT ?field ?value ?count
WHERE {
    (?field ?value ?count)
        luc:facet ('copper'
            '["commodity","state","status"]' 10)
}
ORDER BY ?field DESC(?count)
Performance
O(1) facet counting — pre-built DocValues structures, not document iteration. ~25% indexing overhead for DocValues creation at write time.
ArgumentDescription
queryStringLucene query to scope facets
facetFieldsJSON array of field names
filterOptional JSON filter (same as luc:query)
maxValuesMax values per field (0 = all)
minCountExclude rare values below threshold

Shared Execution

When luc:query and luc:facet appear in the same SPARQL query with matching parameters, they share a single Lucene execution — one index read, consistent snapshot. 

-- UNION pattern: N+M rows, not N*M
SELECT ?entity ?score ?totalHits
       ?field ?value ?count
WHERE {
  { (?entity ?score ?_lit ?totalHits)
        luc:query ('copper' 10) }
  UNION
  { (?field ?value ?count)
        luc:facet ('copper'
            '["state","commodity"]' 10) }
}

Sequence & Inverse Paths

SHACL sh:path expressions traverse multiple predicates or reverse direction. Indexed transparently — the SPARQL consumer never sees the traversal.

Sequence Path
sh:path ( ex:authoredBy ex:name )

Traverses Report → Author → Name at index time. The authorName field contains "Dr Sarah Jones", not a URI.
Inverse Path
sh:path [ sh:inversePath ex:authored ]

Traverses ^ex:authored backwards from Author to Report. Indexes the author URI on the report.
Filter by Author (Sequence Path) 
PREFIX luc: <urn:jena:lucene:index#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?entity ?score ?label
WHERE {
    (?entity ?score) luc:query
        ('*' '{"authorName":["Dr Sarah Jones"]}') .
    ?entity rdfs:label ?label .
}
ORDER BY DESC(?score)

Automatic Index Maintenance

ShaclTextDocProducer listens to every triple change. On add or delete of a relevant predicate or rdf:type, the entire entity document is rebuilt from the base dataset.

Design Choice
Full rebuild per entity change (not incremental field update) ensures correctness for multi-valued fields and sequence paths. For typical entities (<50 triples), rebuild is sub-millisecond.
Graph Support
Reads from MultiUnion(defaultGraph, unionGraph) — entities indexed regardless of whether data is in default or named graphs. 
flowchart TD
    A["Triple Add/Delete"] --> B["DatasetGraphTextMonitor"]
    B --> C["super.add() — base dataset updated"]
    C --> D["ShaclTextDocProducer.change()"]
    D --> E{Relevant predicate?}
    E -->|rdf:type| F["handleTypeChange()"]
    E -->|indexed field| G["rebuildEntityDocuments()"]
    E -->|irrelevant| H["ignore"]
    F --> G
    G --> I["Read all entity triples\nfrom MultiUnion graph"]
    I --> J["Build Entity object\nwith all field values"]
    J --> K["Delete old Lucene doc\nAdd new Lucene doc"]

Proposed Extensions

What's Next

All proposed features are purely additive — new PFs or opt-in parameters. No breaking changes to existing APIs.

Spatial Filtering
Bounding-box filter via Lucene LatLonPoint. Scope search + facets to a geographic region. 
luc:facet ("climate"
  '["category"]'
  '{"bbox":[40,-75,41,-74]}')
DrillSideways
Single-pass combined search + facet collection. When filtering by state=QLD, the state facet still shows counts for all states.
Prerequisite: Entity-per-document Done
Range Facets
Numeric bucket aggregation. Year ranges, depth ranges — using existing IntPoint/LongPoint fields. 
luc:facetRange ("*"
  "year"
  '[2020,2022,2024,2026]')
Hierarchical Facets
Taxonomy drill-down via path-based FacetField. SKOS broader/narrower traversal for category trees.
Suggest / Autocomplete
Type-ahead via Lucene SuggestField + AnalyzingInfixSuggester. 
luc:suggest ("mach" 10)
Bulk Reindexer
SHACL-aware CLI tool. Full reindex without per-triple rebuild overhead. Parallel entity document construction.
